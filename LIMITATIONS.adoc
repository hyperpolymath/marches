= LIMITATIONS — prior art and honest boundaries

Searched 2026-07-07 (web). This is a *first sweep*, not the full
literature review; a systematic pass is still owed before any paper
claim. Nearest neighbours, in order of proximity:

== agda-routing (Daggitt, Zmigrod, van der Stoep, Griffin)

The dominant occupant of "mechanised routing algebra + convergence".
An Agda library formalising Sobrinho-style algebras, Üresin–Dubois
asynchronous fixed-point theory (ITP 2018, JAR 2019), rate-of-convergence
results (ICNP 2018), dynamic asynchronous iterations (JPDC 2022), and
*formally verified convergence of policy-rich DBF protocols* (IEEE/ACM
ToN 2023), including a safe-by-design BGPLite algebra with a verified
simulation argument. https://github.com/MatthewDaggitt/agda-routing

Consequence for marches: convergence-of-algebras is *their* ground.
marches must not re-claim it. The residue: region-crossing as a typing
judgment with unconstructibility, composition-as-types, and estate
integration (attestation, grading). Rung 1+ work must cite them first.

== Kirigami / NV (Alberdingk Thijm, Beckett, Gupta, Walker)

Modular control-plane verification by *cutting* a network into fragments
along user-annotated interfaces; assume-guarantee reasoning; proved sound
and complete w.r.t. monolithic verification (ICNP 2022; ToN 2024). Built
on NV. This occupies "regions with interface annotations, verified
modularly" — but as *post-hoc checking of given configurations*, SMT-based.
marches' differentiation is by-construction: the interface is a type whose
violations are underivable, not a property checked afterwards.

== Others (from training knowledge; verify citations at write-up time)

* *Metarouting / Sobrinho algebras* (Griffin & Sobrinho; Sobrinho 2005) —
  the algebraic language marches speaks; strict inflation is Sobrinho's
  strict monotonicity. marches adds nothing at the algebra level.
* *Gao–Rexford 2001* — the stability conditions the GR module encodes;
  the acyclic-hierarchy assumption is theirs, and remains ASSUMED here.
* *Griffin–Shepherd–Wilfong* — stable paths problem, DISAGREE / BAD GADGET,
  NP-hardness of safety; source of the negative artifact.
* *NetKAT and successors* — typed/equational reasoning for the *data plane*,
  intra-domain; not inter-domain policy composition.
* *SCION* — deployed clean-slate inter-domain architecture with isolation
  domains and Isabelle/Tamarin-verified protocol components; regions exist
  there as infrastructure, not as a typing judgment on policy traces.
* *P4* — loop-free-by-construction data-plane programming; the
  by-construction spirit, different plane.
* *RINA, Nimrod, ICING* — architectural ancestors of regioned routing;
  no mechanised typing discipline.

== Known internal gaps

* The stratified order (`CBetter.ab`) makes crossings worsen *by
  definition of the interface's meaning*. That is the design stance
  (interface = obligation). The load-bearing role of within-region
  strictness is now exhibited: `Marches/Freewheel.lean` (falsifier F2,
  discharged 2026-07-07).
* `Interface.strict` was RETIRED at rung 2. It was recorded-but-unused
  through rungs 0–1 as a candidate dependency for n-region composition;
  building that composition (`Marches/Reassembly.lean`) showed crossing
  maps need no order-preservation — the composite worsens by DAG
  position alone — so the field was deleted rather than kept as a
  proven-unused wart. Route-optimality of crossings is a different axis
  this repo does not claim.
* GR is wired into the trace judgment (`Marches/GRTrace.lean`,
  `Marches/GRHierarchy.lean`, rung 1), but the acyclic
  customer→provider hierarchy remains an ASSUMED hypothesis
  (`Marches.Acyclic`) — `gr_loop_free` is conditional on it, and
  loop-freedom of the typed judgment is *not* protocol convergence
  (agda-routing's ground).
* The region DAG (rung 2, `DiscSys.compose`) consumes the SAME
  `Marches.Acyclic` as the GR hierarchy: an acyclic region DAG is an
  ASSUMED property of a deployment, never proven here. Composite
  loop-freedom is by-construction, still not convergence.
